CSP-safe by design

Build reactive interfaces with signal precision, not framework bulk.

StingJS combines Alpine-style HTML directives with Solid-style fine-grained reactivity. It is small, framework-agnostic, and designed to run in strict Content Security Policy environments without runtime expression evaluation.

counter.html

<div x-data="counter">
  <button x-on:click="dec">-1</button>
  <span x-text="count"></span>
  <button x-on:click="inc">+1</button>
</div>

<script type="module">
  import sting from "stingjs"

  sting.data("counter", () => {
    const [count, setCount] = sting.signal(0)
    return {
      count,
      inc: () => setCount((n) => n + 1),
      dec: () => setCount((n) => n - 1)
    }
  })
</script>

Why teams choose StingJS

You keep control of your DOM, ship less JavaScript, and still get composable state and lifecycle tooling.

Directive-first authoring

Use simple x-* attributes for rendering, events, and binding. Keep logic in plain JavaScript factories.

Fine-grained updates

Signal dependencies are tracked at read time, so updates target exactly what changed.

CSP compatibility

Path-safe directive expressions only. No eval, no new Function, no inline expression engine.

Progressive by default

Mount component roots anywhere in server-rendered HTML. No heavy app shell required.

Core + create flow

Install the runtime with npm i stingjs or scaffold a project with npm create sting@latest.

Tested behavior contract

The repository includes Playwright tests for directives, edge cases, strict CSP fixtures, and scaffold generation.

Get started quickly

Two paths: add StingJS to an existing project, or scaffold a fresh app with the create package.

Install runtime

npm i stingjs

Scaffold app

npm create sting@latest

Run locally

npm install && npm run dev

Need TypeScript from day one? Run npm create sting@latest my-app -- --template typescript --yes.
